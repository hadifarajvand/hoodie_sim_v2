# Data Model: Feature 068

## BaselinePolicy

Represents one paper baseline policy.

Fields:

- `name`: stable registry name.
- `aliases`: optional compatibility names.
- `preference_rule`: baseline-specific action preference.
- `fallback_rule`: behavior when the preferred action is unavailable.
- `randomness`: none, deterministic, or seeded random.

Validation rules:

- Name must resolve from the registry.
- Policy must consume PolicyContext.
- Policy must return only an available action or documented no-choice result.

## PolicyContext

Shared input to baseline policies.

Fields:

- `observation`: policy-facing observation values.
- `legal_action_mask`: available action flags or equivalent mask.
- `trace_history`: optional previous decisions or state.
- `seed` or `rng`: optional source for reproducible random behavior.

Validation rules:

- Policies must not bypass the mask.
- Policies must not read environment internals directly.
- Missing optional fields must not produce hidden behavior.

## DelayCandidate

Candidate action considered by MLEO.

Fields:

- `action_family`: local, horizontal, or vertical.
- `action_id`: concrete action id or alias.
- `available`: whether the candidate is allowed.
- `queue_delay`: queue component when known.
- `transmission_delay`: transmission component when known.
- `compute_delay`: compute component when known.
- `total_delay`: comparable sum when all required components exist.
- `tie_break_key`: deterministic key for ties.
- `fallback_notes`: reason candidate could not be compared.

Validation rules:

- Unavailable candidates are removed before ranking.
- Comparable candidates require total_delay.
- Ties are deterministic.
- Missing fields create visible fallback behavior.

## BaselineFidelityEvidence

Evidence generated by tests or implementation notes.

Fields:

- required baseline names.
- registry resolution status.
- mask compliance status.
- RO seed status.
- BCO balancing status.
- MLEO candidate summary.
- fallback behavior summary.
- controlled differentiation status.
- scope audit status.

Validation rules:

- Evidence must be reproducible.
- Evidence must not depend on generated campaign outputs.
- Evidence must identify unresolved risks.
