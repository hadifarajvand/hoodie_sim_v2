# Phase 1 Gate 4D Run Log: Real Paper State Population

## Purpose of Gate 4D
Gate 4D aims to replace the placeholder zero-state vectors in the full training reproduction campaign with real state vectors generated by the PaperStateBuilder. This ensures that the trainer uses realistic 74-dimensional state vectors (task size one-hot, processing density, wait times, queue lengths, load forecast) instead of all-zero placeholders, enabling meaningful learning in the paper state dimension.

## Confirmed Previous Blocker
The previous blocker was the use of all-zero placeholder state vectors in the paper state dimension (state_dim != 3). Specifically, in `tests/unit/test_full_training_reproduction_campaign_replay_paper_dimensions.py`, the test `test_validate_zero_state_replay` asserted that all state vector rows were zeros (self.assertTrue(all(v == 0.0 for v in row))). This masked the lack of real state computation and prevented validation of the PaperStateBuilder integration.

## Implementation Summary
1. **PaperStateBuilder Integration**: The `DDQNTrainer` in `src/analysis/full_training_reproduction_campaign/trainer.py` now initializes a `PaperStateBuilder` instance (regardless of state_dim) and uses it to compute state vectors when `state_dim != 3` (paper path). The legacy 3D path remains unchanged, using zero placeholders for backward compatibility.

2. **Test Fix**: The stale assertion in `tests/unit/test_full_training_reproduction_campaign_replay_paper_dimensions.py` (line 178-181) was replaced with validation that the state vector:
   - Has length 74
   - Is not all zeros
   - Has a task-size one-hot vector (first 31 elements) summing to 1.0
   - Has the first bin (index 0) active for the default task size (0.0)
   - Has processing density, wait times, queue lengths, and load forecast set to appropriate default values (0.0 for the zero state)

3. **Validation**: All tests in `test_trainer_uses_real_state.py` now pass, confirming:
   - The trainer contains a PaperStateBuilder instance
   - `_compute_state_vector` uses the builder to generate real state vectors
   - `_initial_history` uses real state computation for paper dimensions (not hardcoded zeros)
   - The legacy 3D path remains unchanged (still uses zeros, but the state_builder exists and is conditionally used)

## Exact Stale Test Assertion That Was Fixed
**Before** (in `tests/unit/test_full_training_reproduction_campaign_replay_paper_dimensions.py`):
```python
self.assertTrue(all(v == 0.0 for v in row))
```

**After** (same file):
```python
# Validate real paper state: length 74, non-zero, task size one-hot sums to 1.0, index 0 active for size=0.0
self.assertEqual(len(row), 74)
self.assertFalse(all(v == 0.0 for v in row), "Paper state should not be all zeros")
size_one_hot = row[:31]
self.assertAlmostEqual(sum(size_one_hot), 1.0, places=6)
self.assertAlmostEqual(size_one_hot[0], 1.0, places=6)  # size=0.0 maps to index 0
self.assertAlmostEqual(row[31], 0.0, places=6)  # processing density
self.assertAlmostEqual(row[32], 0.0, places=6)  # private wait time
self.assertAlmostEqual(row[33], 0.0, places=6)  # offload wait time
self.assertTrue(all(v == 0.0 for v in row[34:54]))  # public queue lengths
self.assertTrue(all(v == 0.0 for v in row[54:74]))  # load forecast
```
